# 🚀 Project Roadmap & Future Updates (GitHub Issues)

이 리스트는 향후 서비스 고도화를 위한 핵심 이슈들입니다. 각 항목은 GitHub의 Issue로 등록하여 관리하는 것을 권장합니다.

---

### 📊 [Data & Analysis]
#### #1. 실시간 스피또 잔여 수량 크롤러 고도화 (High)
- **설명**: 동행복권 사이트의 스피또 잔여 수량 테이블을 1시간 단위로 자동 크롤링하여 `data/live_status.json` 업데이트.
- **목표**: 사용자에게 정확한 1등 잔여 정보를 제공하여 '스피또 명당' 신뢰도 확보.

#### #2. 머신러닝 기반 '명당 가중치' 모델 튜닝 (Medium)
- **설명**: 단순 당첨 횟수 외에 유동인구, 판매점 개설 연도, 수동 당첨 비율을 변수로 한 정밀 Luck Index 모델 도입.
- **목표**: 타 서비스와 차별화된 우리만의 '전략적 명당' 지수 산출.

---

### 📱 [Feature & UX]
#### #3. PWA 도입을 통한 푸시 알림 구현 (High)
- **설명**: Progressive Web App 기술을 적용하여 토요일 추첨 시간(20:45) 전 알림 및 당첨 번호 실시간 푸시 발송.
- **목표**: 앱 설치 없이도 사용자 리텐션을 극대화.

#### #4. 실제 사용자 당첨 후기 및 인증 게시판 (Medium)
- **설명**: 사용자가 우리 앱의 번호나 명당 지도를 통해 당첨된 사례를 사진(영수증)과 함께 업로드하는 기능.
- **목표**: '로또리치' 스타일의 사회적 증명(Social Proof)을 통한 신뢰 구축.

#### #5. 카카오톡 공유 API 정식 연동 (Medium)
- **설명**: 현재 alert 방식의 공유를 카카오톡 메시지 템플릿(이미지+버튼) 형식으로 교체.
- **목표**: 바이럴 루프의 전환율 향상.

---

### 📈 [Marketing & SEO]
#### #6. 롱테일 키워드 기반 SEO 콘텐츠 페이지 생성 (Low)
- **설명**: '로또 세금 계산기', '꿈해몽 번호 찾기', '복권 판매점 창업 가이드' 등 검색 유입이 많은 키워드별 전용 랜딩 페이지 구축.
- **목표**: 지도 기능 외 다양한 경로의 유기적 트래픽(Organic Traffic) 확보.

#### #7. 랜딩 페이지 A/B 테스트 자동화 (Low)
- **설명**: '기술적 분석 강조형' vs '감성적 후기 강조형' 헤더 디자인을 무작위로 노출하여 클릭률(CTR) 측정.
- **목표**: 데이터에 기반한 최적의 전환율 도출.

---

### 🔧 [Maintenance & Fix]
#### #8. 전국 판매점 데이터 정기 무결성 검사 (High)
- **설명**: 폐업하거나 주소가 변경된 판매점을 네이버/카카오 API와 대조하여 자동 필터링하는 스크립트 주기적 실행.
- **목표**: "없는 곳이 허다함"과 같은 사용자 불만 근본적 해결.

#### #9. 저사양/구형 기기 레이아웃 렌더링 최적화 (Medium)
- **설명**: S24 등 최신 기기 외에 구형 안드로이드 폰에서도 WebGL 가속을 통한 부드러운 지도 이동 지원.
- **목표**: 광범위한 사용자 층 확보.
