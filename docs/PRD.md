# 📝 Product Requirements Document (PRD): LottoMap PRO

## 1. 제품 비전 (Vision)
"공공데이터의 투명성과 AI의 통찰력을 결합하여, 모든 사용자가 과학적으로 명당을 탐색하고 행운을 유통하는 대한민국 No.1 로또 지능형 플랫폼."

## 2. 핵심 타겟 (Target Audience)
- **전략형 사용자**: 과거 통계와 패턴을 분석하여 번호를 조합하는 숙련자.
- **감성형 사용자**: 꿈이나 운세 등 주관적 기운을 번호와 연결하고 싶은 사용자.
- **편의형 사용자**: 내 주변 명당을 가장 빠르게 찾고 스피또 잔여량을 확인하고 싶은 사용자.

## 3. 핵심 기능 요구사항 (Functional Requirements)

### 3.1 지능형 명당 지도 (Smart Map)
- [ ] 사용자 GPS 기반 실시간 위치 동기화.
- [ ] **Luck Index**: 당첨 횟수가 아닌 '판매량 대비 효율' 지수 표시.
- [ ] **Speetto Live**: 판매점별 스피또 2000/1000/500 실시간 1등 잔여 수량 노출.
- [ ] 네이버 지도 길찾기 및 다이렉트 전화 연결.

### 3.2 AI 꿈해몽 엔진 (Dream AI)
- [ ] 키워드 검색 기반의 꿈 해석 및 연관 번호 추출.
- [ ] 꿈의 기운과 명당의 기운을 결합한 최종 행운 점수 산출.

### 3.3 전략 통계 대시보드 (BI Dashboard)
- [ ] 최다 출현 번호 TOP 10 바 차트.
- [ ] 최근 5회차 당첨금 트렌드 라인 차트.
- [ ] 홀짝/고저/합계 등 12대 핵심 분석 지표 위젯.

### 3.4 사회적 증명 및 공유 (Social & Distribution)
- [ ] 분석 결과를 '행운 카드' 이미지로 생성 및 카카오톡 공유.
- [ ] 실시간 사용자 트래픽 기반 "기운 받아간 사람" 카운터 표시.

## 4. 비기능 요구사항 (Non-functional Requirements)
- **성능**: Lighthouse 기준 모바일 로딩 성능 90점 이상 달성.
- **SEO**: 지역별(시/군/구) 동적 메타 태그 최적화.
- **안정성**: 중복 Key 및 ReferenceError가 없는 무결성 코드.
- **UX**: S24 등 최신 기기에서 레이아웃 겹침 없는 반응형 디자인.
