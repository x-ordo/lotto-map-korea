# 🎨 LottoShrine UX/UI Strategy: Operation 'Dopamine'

본 문서는 LottoShrine PRO의 시장 장악을 위한 사용자 경험(UX) 및 시각적 언어(UI) 폴리싱 전략을 정의합니다.

---

## 1. Core UX Strategy: 'Contextual Flow'
- **Want vs Need 리밸런싱**: 사용자가 '원하는 것(번호 추출)'을 제공하는 맥락 안에서 '필요한 것(통계/데이터)'을 자연스럽게 노출합니다.
- **내비게이션 다이어트**: 6개 이상의 파편화된 탭을 핵심 행동 중심의 **3대 탭**으로 압축합니다.
- **Adaptive UI**: 폰트 사이즈와 대조비를 상향 조정하여 전 세대를 아우르는 **유니버설 가독성**을 확보합니다.

---

## 2. UI Polishing Roadmap

### Phase 1: 아키텍처 정제 (즉시)
- **탭 통합**: [성지], [금고(추출+인사이트)], [담벼락]으로 구조 재편.
- **Header 최적화**: 로또 결과 허브와 상단 네비게이션의 시각적 위계 정리.

### Phase 2: 인터랙션 강화 (단기)
- **Framer Motion 도입**: 컴포넌트 간 전환 시 부드러운 Spring 효과 적용.
- **Haptic Feedback 시뮬레이션**: 번호 생성 및 인증 시 시각적 진동 효과 구현.

### Phase 3: 데이터 시각화 고도화 (중기)
- **Macro-Micro View**: 지도의 광역 뷰(기운 밀집도)와 상세 뷰(매장 정보)의 심리스한 전환.
- **Social Proof Real-time**: "방금 3명이 여기서 번호를 점지받았습니다" 애니메이션 피드.

---

## 3. Design Tokens (Refined)
- **Base Font**: 16px (Min 14px for Silver Users)
- **Emphasis**: `indigo-600` for Action, `yellow-500` for Fortune.
- **Surfaces**: `Glassmorphism` 적용 (화이트/블랙 배경에 80% 불투명도 및 Blur 적용).
